Homepage — Flow Structure Proposal

現在の構造（課題）
クエリを入力すると、関連するシナリオと技術コンポーネントが生成されます。
♀技術起点 ⊠問題起点 | 技術の全体像を把握する

⚠ FAST（技術ツリー直接生成）が消えている。 ユーザーが「まずツリーを見てから考えたい」というクイック探索ニーズに応えられない。

Option A — ３つ目タブとして追加 シンプル
クエリを入力すると、関連するシナリオと技術コンポーネントが生成されます。
♀技術起点 ⊠問題起点 ⚡クイック探索
技術起点→技術特性 Dialog→有望シナリオ選定
問題起点→BN分解 → 解決シナリオ
⚡ クイック探索→技術ツリー（マインドマップ）

✓ 発見しやすい。3つのモードが並列で明快。
✗ 元の3タブに戻る。「クイック探索」が他2つと同格に見えてしまい重みが違う。

Option B — 技術起点のサブオプションとして追加 推奨
クエリを入力すると、関連するシナリオと技術コンポーネントが生成されます。
♀技術起点 ⊠問題起点 | 技術の全体像を把握する 技術ツリーを先に確認する
技術起点（デフォルト）→技術特性 Dialog→有望シナリオ選定
+ 全体像を把握する→クエリレポート→有望シナリオ選定
+ ツリーを先に確認する→技術ツリー（FAST）→有望シナリオ選定

✓ タブは2つのまま。クイック探索は技術起点の「準備ステップ」として自然に位置づけられる。
✓ どちらのオプションも「最終的には有望シナリオ選定へ収束する」という構造を守れる。
✓ 両オプションを同時にオンにすることも可能（ツリー → レポート → 選定）。
✗ 2つのチェックボックスが並ぶため、初見ユーザーに少し説明が必要かも。

Option C — サジェストチップで差別化 軽量
クエリを入力すると、関連するシナリオと技術コンポーネントが生成されます。
♀技術起点 ⊠問題起点 | 技術の全体像を把握する
試してみる： 全固体電池 mRNA治療薬 FAST 光電融合 FAST 炭素回収技術

✓ タブが増えない。FASTチップをクリックするとそのままツリーへ飛ぶ。UIがすっきり。
✗ 「任意のクエリをFASTモードで検索したい」ときの導線が弱い。サジェストにしかFASTバッジが出ない。

まとめ

推奨はOption B。理由：

  • 「技術起点」は常に最終的に有望シナリオ選定へ収束する。ツリー確認もレポート確認もその前の準備ステップとして自然に位置づけられる。
  • タブが2つのまま保たれ、問題起点との対称性が崩れない。
  • 両オプションが同時にオンでも成立する（ツリー → レポート → 選定の順）。
  • 将来的にオプションが増えても同じパターンで拡張できる。
実装イメージ：「技術ツリーを先に確認する」をオンにして Submit → FAST で技術ツリーへ。 ツリーページに「シナリオ探索へ進む →」ボタンを置き、そこから有望シナリオ選定に合流。
